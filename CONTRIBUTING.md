## Contribution Guidelines

### Code of Conduct
- Be respectful and constructive
- Welcome newcomers
- Focus on the idea, not the person
- Give credit where credit is due

### Quality Standards
- **Examples:** Include before/after comparisons
- **Documentation:** Clear, concise, actionable
- **Code:** Follow existing style and patterns
- **Commits:** Descriptive commit messages

### Pull Request Process
1. Fork the repository
2. Create a new branch (`git checkout -b feature/your-feature-name`)
3. Make your changes
4. Test your changes
5. Commit with clear messages
6. Push to your fork
7. Submit a pull request

### Review Process
- We'll review PRs within 1-2 weeks
- We may request changes or clarifications
- Once approved, we'll merge and credit you
